Инструкция установки:
    1 Установить веб-сервер Apache на компьютер (нужен для публикации базы данных);
    2 Опубликовать базу данных на веб-сервер Apache (включая веб и http сервисы, они должны быть установлены);
    3 В случае если сайт находится на отдельном сервере от опубликованной базы данных, необходимо пробросить внешние порты к базе данных и установить SSL сертификат;
    4 Открыть файл настроек settings.json (описание ниже), заполнить адрес опубликованной базы, остальные настройки по желанию;
    5 Добавить в тэг <head> файл стилей UMC-widget.css перед файлом собственных стилей в таком виде <link rel="stylesheet" href="путь к папке/UMC-widget/UMC-widget.css"> (порядок добавления в CMS может отличаться);
    6 Добавить в тэг <body> файл скрипта UMC-widget.js перед файлом собственных скриптов в таком виде <script src="путь к папке/UMC-widget/UMC-widget.js"></script> (порядок добавления в CMS может отличаться).

Настройки:
    общее:
        размеры: задаются в пикселях (px), процентах (%), относительных величинах (rem, em);
        цвета: задаются в HEX системе (#18538E), rgba системе (rgba(172,174,176,0.8)), ключевыми словами (white, red);

    settings:
        addressFile: адрес или путь к файлу send.php (файл, отправляющий запросы к API УМЦ, по умолчанию './send.php');
        address: адрес опубликованной базы данных УМЦ (слэш в конце обязателен);
        timeReserve: время (в минутах), резервируемое для приема по умолчанию (для изменения необходимо проставить число здесь, а также перейти в Администрирование->Учетная политика->Журнал записи и проставить необходимую периодичность планирования).

    style:
        containerClass: класс элемента, в который необходимо поместить виджет (обязательна точка перед именем класса; если виджет является блоком на странице, необходимо выставить "position: static" в поле "widgetPosition");
        widgetPositionPlace: позиция виджета при "position: fixed", может быть 4 позиции("left-top" - по умолчанию, "right-top", "left-bottom", "right-bottom");
        widgetPosition: позиционирование слайдера, может быть "fixed" - по умолчанию или "static" (при фиксированном положении виджет остается на месте при скролле экрана и закрепляется относительно окна браузера, при статическом позиционировании - закрепляется относительно блока-родителя);
        backgroundColor: цвет фона виджета;
        backgroundWidth: ширина виджета, для адаптивности в статической позиции рекомендуется задать ширину в 100%,и изменять ширину блока-родителя. На мобильных телефонах ширина равна 300px. (по умолчанию 300px);
        backgroundMaxHeight: максимальная высота виджета. По умолчанию calc(100vh - 20px), то есть высота экрана - 20px;
        inputColor: цвет фона текстовых полей;
        placeholderFontColor: цвет имени текстовых полей;
        placeholderFontSize: размер шрифта имен текстовых полей;
        inputFontColor: цвет введенного в текстовые поля текста;
        inputFontSize: размер шрифта текста, введенного в текстовые поля;
        listItemColorHover: цвет заголовков выпадающих списков и элементов выпадающих списков при наведении мыши;
        listItemFontColorHover: цвет шрифта элементов выпадающих списков при наведении;
        scrollBarColor: цвет элемента скролла;
        scrollBarColorHover: цвет элемента скролла при наведении;
        textColor: цвет обычного текста виджета;
        buttonColor: цвет кнопок;
        buttonFontColor: цвет шрифта кнопок;
        buttonFontSize: размер шрифта.

Важно:
    1 Виджет появляется при загрузке информации о клиниках, чтобы отображать его при выполнении кого-либо события необходимо выставить display:none для .UMC-widget по умолчанию и display:block при необходимости его появления.
    2 При каждой записи создается новый клиент. Чтобы не накапливались одинаковые клиенты, необходимо произвести настройку БИТ УМЦ: перейти в Администрирование->Настройки обмена с сайтом->Настройки поиска клиентов (Добавить)->Создать (кнопка со знаком плюс)->Добавить необходимые реквизиты клиента, рекомендуется использовать имя, фамилию, номер телефона.
    3 По умолчанию программой выставлено время приема любого врача - 15 минут. Чтобы изменить время приема конктретного сотрудника перейдите в Персонал и расчет зарплаты->Сотрудники, выбираете сотрудника, откройте его, далее Основные услуги->Создать->Показать все->Создать номенклатуру. При создании указываете продолжительность услуги. Далее в окне сотрудника во вкладке основные услуги добавляете созданную номенклатуру и выбираете индивидуальную продолжительность, именно по таким промежуткам будет делиться  рабочее время сотрудника. Внимание, основная услуга у врача должна быть одна или индивидуальная продолжительность у всех основных услуг должна быть одинаковой. Внимание, настройка Основная длительность приема во вкладке Настройки в окне сотрудника не влияет на продолжительность записи с сайта.
